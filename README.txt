📑 Структура набора данных online_shop состоит из взаимосвязанных таблиц, имитирующих реальную платформу электронной коммерции. Каждая таблица представляет собой ключевой аспект бизнеса, такой как товары, заказы, клиенты, поставщики и отзывы. Ниже приведено подробное описание каждой таблицы и её столбцов:


🛍️ Таблица «заказы» ORDERS включает в себя следующие столбцы:
order_id: Уникальный идентификатор для каждого заказа.
order_date: Дата размещения заказа.
customer_id: Ссылка на клиента, разместившего заказ (связана с customersтаблицей).
total_price: Общая стоимость заказа, рассчитанная как сумма всех товаров в заказе.



👩‍💼 Таблица «клиенты» CUSTOMERS включает в себя следующие столбцы:
customer_id: Уникальный идентификатор для каждого клиента.
first_name: Имя клиента.
last_name: Фамилия клиента.
address: Адрес заказчика.
email: Адрес электронной почты клиента (уникальный для каждого клиента).
phone_number: Номер телефона клиента.



🛒 Таблица «Продукты» PRODUCTS включает в себя следующие столбцы:
product_id: Уникальный идентификатор для каждого продукта.
product_name: Название продукта.
category: Категория, к которой принадлежит продукт (например, Электроника, Дом и кухня).
price: Цена продукта.
supplier_id: Ссылка на поставщика, предоставляющего продукт (связанная с suppliersтаблицей).




📦 Таблица «order_items» ORDER_ITEM_ID  включает в себя следующие столбцы:
order_item_id: Уникальный идентификатор для каждого товара в заказе.
order_id: Ссылка на заказ, содержащий элемент (связанный с ordersтаблицей).
product_id: Ссылка на заказываемый продукт (связанная с productsтаблицей).
quantity: Количество заказанного товара.
price_at_purchase: Цена товара на момент заказа.




🏢 Таблица «поставщики» SUPPLIERS  включает в себя следующие столбцы:
supplier_id: Уникальный идентификатор для каждого поставщика.
supplier_name: Название поставщика.
contact_name: Имя контактного лица у поставщика.
address: Адрес поставщика.
phone_number: Номер телефона поставщика.
email: Адрес электронной почты поставщика.




🌟 Таблица «отзывы» REVIEWS включает в себя следующие столбцы:
review_id: Уникальный идентификатор для каждого обзора продукта.
product_id: Ссылка на рассматриваемый продукт (связанная с productsтаблицей).
customer_id: Ссылка на клиента, написавшего отзыв (ссылка на customersтаблицу).
rating: Оценка, данная продукту (от 1 до 5, где 5 — наивысшая оценка).
review_text: Текстовое содержание обзора.
review_date: Дата написания обзора.



💳 Таблица «платежи» PAYMENTS включает в себя следующие столбцы:
payment_id: Уникальный идентификатор для каждого платежа.
order_id: Ссылка на оплачиваемый заказ (привязана к ordersтаблице).
payment_method: Способ оплаты (например, кредитная карта, PayPal).
payment_date: Дата совершения платежа.
amount: Сумма платежа.
transaction_status: статус платежа (например, «Ожидание», «Завершено», «Не удалось»).



🚚 Таблица «отгрузки» SHIPMENTS включает в себя следующие столбцы:
shipment_id: Уникальный идентификатор для каждой поставки.
order_id: Ссылка на отправляемый заказ (связанная с ordersтаблицей).
shipment_date: Дата отправки груза.
carrier: Компания, ответственная за доставку груза.
tracking_number: Номер отслеживания отправления.
delivery_date: Дата доставки груза (если применимо).
shipment_status: статус отправки (например, «Ожидание», «Отправлено», «Доставлено», «Отменено»).
Этот набор данных представляет собой комплексную симуляцию платформы электронной коммерции, охватывающую всё: от заказов клиентов до взаимоотношений с поставщиками, платежей, поставок и отзывов клиентов. Он отлично подходит для практики SQL, изучения реляционных баз данных, а также для выполнения задач анализа данных и машинного обучения.
